# 更新日志

## v1.1.4

### 修复

- 修复后端删除服务器后,前端仍残留「幽灵卡片」(名称来自缓存、所有指标为 0、`last_active=0000-00-00`)、必须刷新整页才能清掉的问题。`komariToNezhaWebsocketResponse` 中的 `km_servers_cache` 仅在 `length === 0` 时填充一次,之后永不更新;Komari 的 `common:getNodesLatestStatus`(每 2 秒一次的 WS tick 数据源)对**已删除**的服务器会直接不返回该 UUID(只有**离线但仍注册**的服务器才会以 `online: false` 出现),所以可以在渲染循环里直接用 statusMap 过滤掉缓存中已不在后端的 UUID,后端删服务器在下一次 WS tick(≤2 秒)立即从前端消失;同时将 `getKomariNodes()` 的调用从「仅缓存为空时」改为「每次 tick」(命中其自带 2 分钟 TTL 时近乎零开销),长期保持 `km_servers_cache` 与后端一致。
- 修正服务器卡片和资产卡片对 `price === 0` / `price === -1` 的语义解读。Komari 后端官方语义是 `price === -1` = **免费**(无意义价格但仍参与展示),`price === 0` = **未配置**(不应作为账单价格显示)。v1.1.3 错误地把两者都当成「免费」类(资产卡片显示「免费/一次性」、服务器卡片走「按量计费」绿色文案)。本版:
  1. 服务器卡片(`billingInfo.tsx`):`-1` 显示绿色「免费」,`0` 不渲染价格行。
  2. 资产卡片(`AssetSummaryWidget.tsx`):`-1` 显示「免费」、`0` 显示「未设置」,两者统一不计入资产汇总。

## v1.1.3

### 修复

- 修复资产卡片中**多年期套餐**的剩余价值被错误地按 1 年计算的问题。Komari 后端 `billing_cycle` 为 1825（5 年）时，主题的 `deriveCycleLabel` 会输出中文 "五年"，但下游的 `parseCycleDays` 只识别阿拉伯数字+年（如 "5y"、"5年"），中文数字会进入兜底逻辑被识别成 365 天。这会导致 `daysLeft / cycleDays` 大幅 > 1 被 `Math.min(1, ...)` 截断成 1，所有 5 年期机器的剩余价值都被显示为原价，相同套餐的多台服务器看上去剩余价值"完全相同"，无法体现到期日差异。本版双管齐下：
  1. `parseCycleDays` 增加中文数字（一/二/两/三/.../十、十X、X十、X十Y）+ 年/月/天的识别。
  2. `deriveCycleLabel` 的 `CYCLE_MAP` 改为输出阿拉伯数字（"2年" / "3年" / "5年"），消除歧义来源。
- 修复 Komari 后端 `price = -1` 的服务器在资产卡片里被误显示为「按量」的问题。Komari 后端 `formatPrice` 函数（`api/public/mjpeg.go:1368-1374`）的官方语义是 `price < 0` = **"免费/一次性"**、`price == 0` = **"免费"**，Komari 中并不存在"按量计费"概念。本版把 `-1` 改为显示「免费/一次性」并归入 `isFree` 分类，与 Komari 自带卡片显示对齐，过滤免费资产、未换算计数等逻辑也会正确处理这类机器。

### 内部

- 抽出 `parseChineseNumeral` 工具函数解析 1-99 的中文数字。
- `AssetSummaryWidget` 的 `isUsageBased` 字段在 Komari 语义下永远为 false，保留字段是为了避免破坏类型契约；过滤/计数逻辑通过 `isFree` 兜底。

## v1.1.2

### 修复

- 修复资产卡片在「默认账单货币」被显式设为 CNY/JPY 等具体货币（非"跟随后端"）时，会忽略 v1.1.1 引入的 tags 内嵌 `<货币>` 元标签，把日元服务器误识别为默认货币的问题。原因是 `komariToNezhaWebsocketResponse` 转换出的 `NezhaServer` 对象未透传 Komari 后端的 `tags` 和 `currency` 字段，导致资产卡片调用 `resolveThemeBillingCurrency` 时 tag 这一层失效。本版透传这两个字段，资产卡片现在能与服务器卡片金额前缀一致地识别 `<JPY>` 等元标签。

### 内部

- `NezhaServer` 类型新增可选字段 `tags?: string`、`currency?: string`，作为后端原始数据的兜底通道。

## v1.1.1

### 修复

- 修复人民币计价的服务器金额前缀显示成 `CNY 200.20` 文字而非 `¥200.20` 符号的问题。原本为消除人民币/日元的 `¥` 歧义，CNY 一律渲染为文字前缀；但对于"绝大多数服务器是 CNY、少数是 JPY"的部署，让多数派背消歧义负担反而更不直观。改为默认 CNY 用 `¥`、JPY 仍用 `JPY ` 文字（消歧义负担落在少数派）。

### 新增

- 主题设置新增「人民币显示风格」下拉，可选 `¥` 或 `CNY`，默认 `¥`。日元为主的部署可切到 `CNY` 文字风格。
- 服务器货币现在支持通过 Komari 后端 tags 字段内嵌元标签配置：在 tags 里加 `<JPY>` / `<USD>` / `<CNY>` 等 ISO 代码标签，主题会自动识别并应用，且这些标签不会在前端 UI 中显示。例如把某台日元结算的服务器 tags 从 `Tokyo; KDDI` 改成 `Tokyo; KDDI; <JPY>` 即可，无需在主题面板填 JSON。
- 货币优先级链：主题面板 JSON 覆盖 > tags 内嵌 `<CURRENCY>` > 默认账单货币 > Komari 后端 `currency` 字段。

### 内部

- 抽出 `parseTagMetadata` 函数同时返回清洗后的 tags 文本和 currency 元数据，复用现有的 `<color>` 标签解析机制。

## v1.1.0

### 修复

- 修复服务器随机显示离线的问题。在线判定原本依赖访客本地时钟与服务器时钟的差值（≤30 秒视为在线），当访客设备时钟比 Komari 服务端快 27 秒以上时会触发误判。Caddy 反代下、移动端切前后台后、Windows 长时间未同步时间都容易复现。改为优先使用 Komari 后端权威的 `online` 字段（基于 WebSocket 连接 + presence TTL），仅在该字段缺失时才回退到时间差判断，从根本上消除时钟偏差影响。
- 修复 `WebSocketProvider` 的 `setInterval` 在卸载时未清理的问题，避免组件多次挂载或热更新时产生重复轮询。
- 为后台数据轮询补上 `.catch`，单次请求失败不再产生未处理的 Promise rejection，下一轮自动恢复。

### 新增

- 主题设置中新增「访客胶囊颜色」下拉选项，可在 8 种主色（`blue / green / purple / pink / orange / red / cyan / amber`）之间切换，亮/暗模式自动配套。
- 主题设置中新增「资产卡片颜色」下拉选项，色板与访客胶囊一致，可独立配置；触发按钮、面板标题、金额、汇率、Hover 效果、交易输入框 focus 高亮统一切换主色。

### 内部

- 抽出 `src/lib/theme-colors.ts` 集中管理上述两个组件的色板配置。
- `tailwind.config.js` 增加 `safelist`，防止未来重构时动态色 class 被 JIT 摇树。

## v1.0.6

- 主题设置中新增“显示资产卡片”和“资产卡片默认币种”，开启后显示右上角资产统计浮窗、汇率换算、排序、免费资产过滤和交易计算。

- 资产卡片的单台服务器价格读取主题合并后的账单数据，会匹配“默认账单货币”和“服务器货币覆盖”配置。

### 修复

- 修复 Komari 后端货币填全角 `￥` 时资产卡片无法识别的问题，现在默认按 CNY 参与资产换算。
- 修复 Komari 后端货币填裸 `¥` 时资产卡片仍提示无法换算的问题；裸 `¥/￥` 默认按 CNY，JPY 需使用 `JPY`、`JP¥`、`日元` 或单机覆盖。
- 调整访客胶囊为蓝色样式，并移除右上角关闭按钮。
